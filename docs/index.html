QuBins

QuBins — the place for your QuBits: prebuilt quantum compartments, pick one, run your Qiskit notebook on (my)binder or as a container (“bin”).

GitHub repo

Quick start

Run the latest Qiskit (…) in your browser via Binder — no install:

Launch latest-small in Binder Launch latest-xl in Binder

Or pull with Docker: 

docker run --rm -p 8888:8888 ghcr.io/janlahmann/qiskit:latest-small

small = lean (qiskit + aer + ibm-runtime). xl = broad, with addons + scientific stack + nbgitpuller + single-file ?fromURL= launcher. Full details in the README.

    The launch URL (/launch/?…) is a thin JavaScript redirector hosted here on GitHub Pages. It builds the mybinder URL on the fly, which means encoding tweaks land in one place — every badge in the wild keeps working.

    Want to embed a launch URL for a specific notebook in a repo or a raw .ipynb URL? Use the launch URL generator above and put its output as the link target on either of the badges here.

    How it works

    Dockerfile is parameterised by Qiskit version × flavor and installs the matching versions/<target>/requirements.txt. A GitHub Actions workflow builds per-arch (amd64 + arm64), gates on a Trivy scan, publishes the multi-arch manifest to GHCR with a SLSA provenance attestation, signs it with cosign, and force-pushes a tiny stub branch that mybinder.org consumes to skip the dependency-tree rebuild.

    A daily cron rebuilds everything against the current base image to absorb upstream CVE fixes. Dependabot watches the docker base, GitHub Action versions, and the latest Qiskit minor's pip pins.

    More in the repository README.

    All in-browser launches here run on mybinder.org, a free service provided by the Binder project / GESIS / 2i2c and federated partners. QuBins is “just” the curated container images they pull; please be patient on cold starts and don’t hammer the service. Consider donating to Project Jupyter if you find Binder useful.
